## Closed Bugs
[x] Nyuidj from TGTT have advantage on Wis saves (Dual Mind racial trait, registered as a `save:advantage:wis` modifier), but pressing Ctrl/Cmd to add disadvantage rolled with disadvantage instead of canceling to a normal roll per RAW. Root cause was in `_rollD20` in `js/charactersheet/charactersheet.js` (~L8500): the four roll callsites (saves, ability checks, skill checks, attacks) precomputed a `mode` string from passive state alone, then `_rollD20` *unconditionally* overrode that mode with the event modifier keys (`shiftKey` → `"advantage"`, `ctrlKey/metaKey` → `"disadvantage"`) — so a passive `mode = "advantage"` silently became `"disadvantage"` on Ctrl, with no chance to cancel. Symmetric defect: passive disadvantage + Shift also stacked instead of canceling. Fixed by extracting a single pure helper `CharacterSheetClassUtils.resolveD20Mode({stateAdvantage, stateDisadvantage, event})` that combines the two sources symmetrically — `adv = stateAdvantage || event?.shiftKey`, `dis = stateDisadvantage || event?.ctrlKey || event?.metaKey`, then `adv && dis → "normal"`, else single side wins. `_rollD20` now takes optional `stateAdvantage`/`stateDisadvantage` booleans; when either is provided it routes through the helper (cancel-aware path), otherwise it preserves the legacy event-only path so `_rollInitiative` and the public `rollD20` pass-through API are unchanged. The four affected callsites — `_rollAbilityCheck`, `_rollSavingThrow`, `_rollSkillCheck`, `_rollAttack` — were updated to drop their local `let mode; if (hasAdv && !hasDis) mode = …` blocks and pass the booleans directly, and each now also computes an effective `(stateAdvantage || event.shiftKey, stateDisadvantage || event.ctrlKey/metaKey)` pair for `_getActiveStateEffectLabel` so the dice-result title shows `[States: Adv+Disadv cancel]` whenever cancellation actually fires (including the previously invisible "user keys cancel passive state" case). Helper lives on `CharacterSheetClassUtils` so tests don't need to load the full DOM-coupled `CharacterSheetPage` module. Covered by new `test/jest/charactersheet/CharacterSheetD20ModeResolution.test.js` (16 tests across one describe): every (stateAdv, stateDis, shiftKey, ctrlKey, metaKey) combination that matters — empty input, state-only adv/dis, event-only adv/dis (shift, ctrl, meta), the Nyuidj fix (state adv + ctrl → normal), the Mac variant (state adv + meta → normal), the symmetric case (state disadv + shift → normal), agreeing-stack cases (state adv + shift stays adv; state disadv + ctrl stays disadv), three-way cancellation (both state sides + shift still cancels), explicit-false equivalence, and an irrelevant-key probe (alt key ignored). The 19 existing TGTT Nyuidj tests pass unchanged. Full charsheet suite: 122 / 8185 green.

(B) **Surface rewire** — `charactersheet-spells.js` now routes every cantrip-counting surface through `countPlayerChosenCantrips`: the spell tab status bar, `_checkSpellLimits` (Add-Spell modal cap), `_renderStandardSpellLayout`, `_renderSpellbookLayout`, and `_renderSpellTrackingUI`. The thin `CharacterSheetSpells.isPlayerChosenSpell` wrapper now delegates to ClassUtils with the new (orphan-excluding) semantics. (C) **Orphan visibility group** — both layout renderers partition cantrips into `attributed + featureGranted` (rendered in their normal level-0 group, counted toward cap) and `orphan` (rendered in a new `_appendOrphanCantripsGroup` "Other Cantrips" group with a muted "(unattributed — does not count toward cap)" subtitle and a tooltip explaining how to re-attribute). For Annabel this now correctly shows `Cantrips (3/4)` everywhere with Mending + Moment to Think listed under "Other Cantrips". Tests: new `CharacterSheetCantripPreparedCounts.test.js` (helper-only contract: 23 cases including Annabel-shaped fixture, multiclass byClass breakdown, prepared-count semantics — cantrips excluded, spellbook-only spells excluded, alwaysPrepared counted); two existing `CharacterSheetSpells.test.js` cases that asserted the old buggy "orphans count" behaviour rewritten to assert the new canonical contract. Full charsheet suite: 123 / 8207 green.
